Die vollständige Anleitung zum Seitenladen von Apps auf Wear OS mit ADB Wireless

Der Wear OS Google Play Store enthält Tausende von Anwendungen, die speziell für Smartwatch-Bildschirme entwickelt wurden. Es kann jedoch vorkommen, dass Sie eine Anwendung installieren möchten, die nicht offiziell im Uhrengeschäft erhältlich ist. Dabei kann es sich um eine vereinfachte Version einer Messaging-App, eine ältere Version eines Gesundheits-Trackers, ein Dienstprogramm wie einen benutzerdefinierten Dateimanager oder eine Beta-Anwendung handeln, die Sie selbst testen. Dieser Vorgang wird aufgerufen Seitenbeladung.

Da Smartwatches normalerweise nicht über Standard-USB-Anschlüsse zum direkten Anschluss an Ihren PC verfügen, nutzt der Sideload-Vorgang die integrierte drahtlose Android Debug Bridge (ADB)-Schnittstelle von Android. In dieser Anleitung behandeln wir die Voraussetzungen, Konfigurationsschritte und Terminalbefehle, die zum Installieren eines kompatiblen APK auf Ihrer Wear OS-Uhr erforderlich sind.

Voraussetzungen: Vorbereitung

Stellen Sie vor dem Ausführen der Sideload-Befehle sicher, dass Sie die folgenden Komponenten zusammengestellt haben:

  • Eine Android APK-Datei: Laden Sie die APK-Datei herunter, die Sie installieren möchten. Stellen Sie sicher, dass die App-Architektur kompatibel ist (normalerweise 32-Bit-ARM für ältere Uhren oder 64-Bit-ARM für neuere Wear OS 4- und 5-Uhren).
  • ADB auf Ihrem PC installiert: Sie benötigen Android SDK-Plattformtools. Wenn Sie sie nicht haben, laden Sie das offizielle ZIP-Archiv von der Android-Entwicklerseite herunter, extrahieren Sie es und fügen Sie den Ordnerpfad zu den Umgebungsvariablen (PATH) Ihres Systems hinzu.
  • Lokaler Netzwerkzugriff: Sowohl Ihre Uhr als auch Ihr Computer müssen mit demselben Wi-Fi-Netzwerk verbunden sein.

Sicherheitswarnung

Laden Sie APK-Dateien nur von seriösen Quellen (wie APKMirror) herunter oder erstellen Sie sie selbst. Das Querladen bösartiger Apps kann Ihre Gesundheitsdaten, Passwörter und die Akkulaufzeit Ihrer Smartwatch gefährden.

Schritt 1: Aktivieren Sie Entwickleroptionen und WLAN-Debugging

Bevor Sie Befehle senden können, muss Ihre Uhr das Debuggen von Verbindungen zulassen. (Eine detaillierte Aufschlüsselung finden Sie in unserem Leitfaden unter So aktivieren Sie Entwickleroptionen unter Wear OS).

  1. Offen Einstellungen > System (oder Über Watch) > Software-Info.
  2. Klopfen Build-Nummer 7 Mal, bis die Entwickleroptionen freigeschaltet sind.
  3. Gehe zu Einstellungen > Entwickleroptionen, dann aktivieren ADB-Debugging.
  4. Scrollen Sie nach unten und wählen Sie aus Drahtloses Debuggen. Schalten Sie es ein und notieren Sie sich die angezeigte IP-Adresse und den Port (z. B. 192.168.1.15:5555).

Schritt 2: Verbinden Sie Ihren Computer mit der Uhr

Öffnen Sie ein Terminalfenster (Terminal unter macOS/Linux oder Eingabeaufforderung/PowerShell unter Windows) und führen Sie die Verbindungsprotokolle aus.

Für moderne Uhren (Wear OS 4 & 5 / Pixel Watch 2+ / Galaxy Watch 6+):

Diese Geräte verwenden eine sichere drahtlose Kopplung. Auf der Uhr, unten Drahtloses Debuggen, klopfen Neues Gerät koppeln. Verwenden Sie den auf dem Bildschirm Ihrer Uhr angezeigten Kopplungsanschluss und Kopplungscode:

ADB-Paar [IP_ADDRESS]:[PAIRING_PORT]

Geben Sie den Kopplungscode ein, wenn Sie dazu aufgefordert werden. Führen Sie nach der Kopplung den Verbindungsbefehl über den Standardport aus dem vorherigen Bildschirm aus:

adb connect [IP_ADDRESS]:[CONNECTION_PORT]

Für ältere Uhren (Wear OS 3 und niedriger):

Wenn Pairing-Codes nicht unterstützt werden, stellen Sie eine direkte Verbindung über Port 5555 her:

adb connect [IP_ADDRESS]:5555

Schauen Sie auf den Bildschirm Ihrer Uhr und tippen Sie auf das Häkchen, um die Debugging-Autorisierung von Ihrem Computer aus zuzulassen.

Schritt 3: Laden Sie die APK-Datei von der Seite

Sobald die Verbindung hergestellt ist, überprüfen Sie, ob die Verbindung aktiv ist, indem Sie die Liste der verbundenen Geräte überprüfen:

ADB-Geräte

Die IP und der Port Ihrer Uhr sollten zusammen mit dem Status angezeigt werden Gerät. Führen Sie nun den Installationsbefehl aus. Verwenden Sie Anführungszeichen oder geben Sie den absoluten Pfad an, wenn der Dateiname Leerzeichen enthält:

adb install „C:\Benutzer\Name\Downloads\app_name.apk“

Abhängig von der Dateigröße und der WLAN-Geschwindigkeit kann die Übertragung zwischen 10 Sekunden und einer Minute dauern. Sobald der Vorgang abgeschlossen ist, wird das Terminal angezeigt Erfolg.

Schritt 4: Optimieren der App-Benutzeroberfläche (optional, aber empfohlen)

Da viele mobile APKs für rechteckige Smartphone-Bildschirme konzipiert sind, können sie auf runden Smartwatch-Bildschirmen übergroß oder abgeschnitten wirken. Sie können dies beheben, indem Sie die Anzeigedichte (DPI) des Geräts skalieren:

Befehl Aktion Nutzungsszenario
ADB-Shell-WM-Dichte Aktuelle Anzeigedichte abfragen Überprüfen Sie die Standardeinstellungen, bevor Sie Änderungen vornehmen
ADB-Shell-WM-Dichte 240 Elemente verkleinern (DPI verringern) Verkleinert den Text und passt auf abgeschnittene Schaltflächen
ADB-Shell-WM-Dichte zurückgesetzt Anzeigeeinstellungen zurücksetzen Stellen Sie die Uhr auf die werkseitige Standardskalierung zurück

Zum Beispiel, wenn Sie laufen ADB-Shell-WM-Dichte 240werden die UI-Elemente der Uhr verkleinert, sodass Sie auf Schaltflächen klicken können, die sich zuvor außerhalb des Bildschirms an den runden Anzeigerändern befanden. Sobald Sie über die Einrichtungsbildschirme der App navigiert sind, können Sie sie mit zurücksetzen ADB-Shell-WM-Dichte zurückgesetzt.

So deinstallieren Sie seitlich geladene Apps

Wenn Sie die Anwendung nicht mehr benötigen, können Sie sie direkt von der Uhr deinstallieren, indem Sie das App-Symbol in der App-Schublade gedrückt halten. Alternativ können Sie es auch mit ADB deinstallieren. Suchen Sie zunächst den Paketnamen der App:

ADB-Shell-PM-Listenpakete -3

Suchen Sie die Kennung der App und führen Sie dann den Deinstallationsbefehl aus:

adb deinstalliert package.name.here

Wenn Sie fertig sind, navigieren Sie auf Ihrer Uhr zurück zu den Entwickleroptionen und deaktivieren Sie beide Drahtloses Debuggen Und ADB-Debugging um die Akkulaufzeit zu verlängern und Ihre Verbindung sicher zu halten.
